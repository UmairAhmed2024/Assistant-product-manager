# Adding your project files

This folder is served directly by the website — anything you put here can
be linked to or downloaded from a project card on the site.

## Current contents

    /public/projects/
      ├── ace-union-product-dashboard-brd.docx        <- real downloadable file
      ├── ace-union-product-dashboard-brd-preview.jpg <- thumbnail shown on the card
      └── README.md                                    <- this file

The preview image was generated by converting page 1 of the .docx to PDF
(LibreOffice) and rendering it to an image (pdftoppm) — so it's an actual
snapshot of the document, not a stock graphic. You can do the same for any
future Word/PDF file:

    soffice --headless --convert-to pdf yourfile.docx
    pdftoppm -jpeg -r 150 -f 1 -l 1 yourfile.pdf preview
    # resize if needed:
    convert preview-1.jpg -resize 900x -quality 88 yourfile-preview.jpg

## Adding a new project

1. Drop the file itself in here — PDF, PPTX, DOCX, PNG, ZIP, whatever it is.
2. (Optional but recommended) generate or add a preview thumbnail image
   using the steps above, or a screenshot/mockup image, and drop that in
   here too.
3. Open `lib/portfolio-data.ts`, find the `projects` array, and add a new
   entry (or edit an existing placeholder one):
   - `title`, `category`, `description`, `tags` — plain text, edit freely
   - `image` — path to the thumbnail, e.g. `/projects/your-preview.jpg`
     (set to `null` to show the default placeholder icon instead)
   - `links.live` / `links.caseStudy` / `links.github` — set any of these
     to a URL, or leave `null` if not applicable
   - `file.path` — `/projects/your-file-name.ext` (note: no "/public" in
     the path — Next.js serves everything in /public from the site root)
   - `file.label` — text on the download button, e.g. "Download BRD (DOCX)"

That's it — the card, image, download button, and links all update
automatically from that one data file. No other code needs to change.
